# 08 — Internacionalización (i18n)

## Estrategia

La internacionalización está implementada con **@ngx-translate/core** y un loader HTTP personalizado. Los archivos de traducción se sirven como assets estáticos y se cargan bajo demanda al cambiar de idioma, sin necesidad de recompilar la aplicación.

La aplicación soporta dos idiomas:

| Código | Idioma | Tamaño del archivo |
|--------|--------|-------------------|
| `es` | Español | ~82 KB |
| `en` | Inglés | ~85 KB |

El idioma por defecto es **español** (`es`). El sistema detecta automáticamente el idioma del navegador del usuario y lo respeta si está disponible.

---

## Configuración

### Registro en `app.config.ts`

```typescript
TranslateModule.forRoot({
  defaultLanguage: 'es',
  loader: {
    provide:    TranslateLoader,
    useFactory: createTranslateLoader,
    deps:       [HttpClient]
  }
})
```

### Loader HTTP Personalizado

**Archivo**: `src/app/i18n/translate-loader.ts`

```typescript
export class HttpTranslateLoader implements TranslateLoader {
  constructor(private http: HttpClient) {}

  getTranslation(lang: string): Observable<any> {
    return this.http.get(`/assets/i18n/${lang}.json`);
  }
}
```

Al usar un loader HTTP en lugar del `TranslateHttpLoader` por defecto, se mantiene control total sobre la URL de los archivos y es posible añadir lógica adicional (versionado, fallback, etc.) sin cambiar la configuración global.

## I18nService

**Archivo**: `src/app/services/i18n/i18n.ts`

Wrapper sobre `TranslateService` que añade persistencia del idioma seleccionado en `localStorage` y encapsula la inicialización automática.

```typescript
@Injectable({ providedIn: 'root' })
export class I18nService {
  get current(): 'es' | 'en' { ... }   // Idioma activo

  use(lang: 'es' | 'en'): void         // Cambia el idioma y lo persiste en localStorage
  instant(key: string, params?: any): string  // Traducción síncrona por clave
  getLangs(): readonly string[]         // ['es', 'en']
}
```

### Lógica de Inicialización

Al instanciarse el servicio, el idioma se determina con la siguiente prioridad:

```
1. Valor guardado en localStorage['lang']  →  Si es 'es' o 'en', se usa directamente
2. Idioma del navegador (getBrowserLang()) →  Si coincide con 'es' o 'en', se usa
3. Fallback                                →  'es' (español por defecto)
```

El cambio de idioma mediante `use()` actualiza `TranslateService` y persiste la preferencia en `localStorage` para que se restaure en la siguiente sesión.

## Estructura de Claves

Los archivos JSON están organizados por dominio funcional. Cada clave de primer nivel agrupa las traducciones de un área de la aplicación:

| Namespace | Área |
|-----------|------|
| `brand` | Nombre y datos de la marca |
| `nav` | Navegación global (menú, links, logout) |
| `auth` | Flujos de autenticación (login, registro, errores) |
| `home` | Landing page |
| `common` | Acciones reutilizables (guardar, cancelar, eliminar, confirmar) |
| `account` | Perfil y configuración del usuario |
| `store` | Tienda de productos |
| `myPurchases` | Historial de compras |
| `mgmt` | Módulos de gestión (etiquetas de menú) |
| `product` | Módulo de gestión de productos |
| `clients` | Módulo de gestión de clientes |
| `sales` | Módulo de gestión de ventas |
| `zones` | Módulo de gestión de zonas |
| `authorities` / `authority` | Módulo de autoridades |
| `bribes` | Módulo de sobornos |
| `decisions` | Módulo de decisiones |
| `topics` | Módulo de temáticas |
| `partner` | Módulo de socios |
| `distributors` | Módulo de distribuidores |

## Uso en Componentes

### Pipe `translate` en templates

```html
<h1>{{ 'nav.store' | translate }}</h1>
<button>{{ 'common.save' | translate }}</button>
```

Con parámetros de interpolación:

```html
<p>{{ 'myPurchases.stats.total' | translate:{ count: total() } }}</p>
```

El pipe `translate` debe incluirse en el array `imports` del componente standalone:

```typescript
@Component({
  standalone: true,
  imports: [TranslateModule, CommonModule, ...],
})
```

### Desde TypeScript

```typescript
private tr = inject(TranslateService);

// Traducción síncrona (el archivo debe estar ya cargado)
const msg = this.tr.instant('auth.login.invalidCredentials');

// Traducción asíncrona (garantiza que el archivo esté cargado)
this.tr.get('common.success').subscribe(text => { ... });
```

### Cambio de Idioma

```typescript
private i18n = inject(I18nService);

toggleLang() {
  const next = this.i18n.current === 'es' ? 'en' : 'es';
  this.i18n.use(next);
}
```

## Agregar Nuevas Traducciones

Para agregar textos de una nueva funcionalidad:

1. Identificar el namespace correspondiente (o crear uno nuevo si es un módulo nuevo).
2. Agregar las claves en `src/assets/i18n/es.json`.
3. Agregar las traducciones equivalentes en `src/assets/i18n/en.json`.
4. Usar la clave en el template con el pipe `translate`.

**Ejemplo** — Añadir textos para un módulo de reportes:

```json
// es.json
{
  "reports": {
    "title": "Reportes",
    "generate": "Generar Reporte",
    "empty": "No hay datos para el período seleccionado",
    "export": {
      "csv": "Exportar CSV",
      "pdf": "Exportar PDF"
    }
  }
}

// en.json
{
  "reports": {
    "title": "Reports",
    "generate": "Generate Report",
    "empty": "No data available for the selected period",
    "export": {
      "csv": "Export CSV",
      "pdf": "Export PDF"
    }
  }
}
```

```html
<h2>{{ 'reports.title' | translate }}</h2>
<button>{{ 'reports.export.pdf' | translate }}</button>
```

## Consideraciones

**Carga bajo demanda**: `ngx-translate` descarga el archivo del idioma seleccionado la primera vez que se necesita. Si el usuario cambia de idioma a `en` por primera vez, el loader hace `GET /assets/i18n/en.json` en ese momento. Las cargas subsiguientes al mismo idioma usan la versión en caché.

**Tamaño de los archivos**: los archivos de ~82-85 KB cubren toda la aplicación. Al ser assets estáticos, la CDN de Vercel los sirve con alta eficiencia desde el edge más cercano al usuario.

**`instant()` vs `.get()`**: `instant()` retorna la clave sin traducción si el archivo de idioma no se cargó aún. En componentes que se inicializan antes de que el loader complete la petición HTTP, usar `.get()` (Observable) o `translate` pipe (que gestiona la asincronía automáticamente) es más seguro.
